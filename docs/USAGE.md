# Руководство пользователя

Полная документация по использованию AI Code Telegram Bot.

---

## Что умеет бот

- **Текст** — отправь любой вопрос или задачу, выбранный AI-провайдер ответит
- **Голос** — отправь голосовое сообщение, бот распознает речь и ответит текстом + голосом
- **Фото** — отправь скриншот или фото документа, бот извлечёт текст (OCR) и обработает
- **Документы** — отправь файл, бот прочитает содержимое и передаст AI-провайдеру
- **Файлы от AI** — скриншоты, CSV и другие файлы, созданные провайдером, автоматически отправляются в чат
- **Длинные ответы** — вместо спама из 10+ сообщений: первый чанк + `.md` файл с полным ответом
- **Провайдеры** — можно переключаться между `claude` и `codex`

## Команды

| Команда | Описание | Кто может |
|---------|----------|-----------|
| `/start` | Приветствие и информация | Все |
| `/help` | Справка и список команд | Все |
| `/new` | Новая сессия (сброс контекста) | user, admin |
| `/cancel` | Отменить текущий запрос | user, admin |
| `/provider` | Показать или сменить AI-провайдера | user, admin |
| `/model` | Показать модели текущего провайдера | user, admin |
| `/model <имя>` | Переключиться на конкретную модель | user, admin |
| `/reasoning` | Показать reasoning текущего провайдера | user, admin |
| `/reasoning <уровень>` | Переключить reasoning | user, admin |
| `/voice` | Включить/выключить голосовые ответы | user, admin |
| `/status` | Текущее состояние (провайдер, модель, reasoning, сессия, голос) | user, admin |
| `/usage` | Твоя статистика за сегодня | user, admin |
| `/stats` | Статистика всех пользователей | admin |

## Типы ввода

### Текстовые сообщения

Просто напиши вопрос или задачу в чат. Активный AI-провайдер обработает её и ответит.

Примеры:
- «напиши функцию сортировки на Python»
- «объясни что делает этот код»
- «исправь баг в файле main.py»

### Голосовые сообщения

Отправь голосовое сообщение — бот распознает речь (faster-whisper), передаст текст AI-провайдеру и ответит:
- Текстом в чат
- Голосовым сообщением (если включён голосовой режим через `/voice`)

Цепочка обработки: 🎤 OGG → ffmpeg → WAV → faster-whisper → текст → AI → ответ → edge-tts → голос

### Фотографии

Отправь фото с подписью или без:
- Бот извлечёт текст через OCR (tesseract)
- Если есть подпись — использует как задачу
- Передаст всё AI-провайдеру для обработки

Сценарии:
- Скриншот ошибки + «исправь» → анализ и решение
- Фото документа + «извлеки текст» → распознанный текст
- Код с экрана + «что делает этот код» → объяснение

### Документы

Отправь файл (`.py`, `.txt`, `.json`, `.csv` и др.) — бот прочитает содержимое и передаст AI-провайдеру.

## Навигация и bash

AI-провайдер может выполнять bash-команды прямо в чате:

```
ls                              — список файлов
cd my-project                   — перейти в проект
mkdir new-project && cd new-project — создать и перейти
pwd                             — текущая директория
git init / git status / ...     — git команды
```

Или на естественном языке:
- «создай папку my-app и перейди в неё»
- «покажи содержимое текущей директории»
- «найди все Python файлы в проекте»

## Провайдеры и модели

Сначала можно выбрать провайдера:

```
/provider claude
/provider codex
```

Потом переключай модель командой `/model <имя>`.

Для `claude` доступны:

| Модель | Особенности | Когда использовать |
|--------|------------|-------------------|
| `haiku` | Быстрая, дешёвая | Простые вопросы, мелкие задачи |
| `sonnet` | Баланс скорости и качества | По умолчанию. Большинство задач |
| `opus` | Максимальное качество | Сложные задачи, архитектура, анализ |

Для `codex` сейчас доступны:

| Модель | Когда использовать |
|--------|--------------------|
| `gpt-5.3-codex` | Codex-ориентированный режим для кода |
| `gpt-5.4` | Текущий дефолт и основной универсальный вариант |
| `gpt-5.2-codex` | Более ранний codex-ориентированный режим |
| `gpt-5.1-codex-max` | Максимальный упор на качество кода |
| `gpt-5.2` | Общий режим без codex-специализации |
| `gpt-5.1-codex-mini` | Более лёгкий и быстрый codex-вариант |

Уровни reasoning для `codex`:

| Уровень | Что значит |
|---------|------------|
| `low` | Быстрее, меньше глубина reasoning |
| `medium` | Баланс скорости и глубины |
| `high` | Больше глубины для сложных задач |
| `xhigh` | Extra High, максимальная глубина reasoning |

Проверить и сменить reasoning можно командами:

```text
/reasoning
/reasoning xhigh
```

Модели и reasoning по умолчанию настраиваются в `.env` через `CODEX_MODELS`, `CODEX_DEFAULT_MODEL`, `CODEX_REASONING_LEVELS` и `CODEX_DEFAULT_REASONING_EFFORT`.

## Роли и лимиты

| Роль | Что может | Лимиты |
|------|-----------|--------|
| `admin` | Всё + `/stats` | Без ограничений |
| `user` | Текст, голос, фото, документы | Дневной лимит сообщений |
| `readonly` | Только чтение ответов | Дневной лимит сообщений |

Текущий лимит можно посмотреть командой `/usage`.

## Сессии и контекст

- Claude и Codex сохраняют контекст разговора в рамках сессии
- При смене провайдера предыдущая сессия сбрасывается
- `/new` — сброс сессии (начать с чистого листа)
- `/status` — показать ID текущей сессии

Когда сбрасывать сессию:
- Переключаешься на другую тему
- Контекст стал слишком большим
- Claude «запутался» в предыдущих сообщениях
- Ты переключаешься между `claude` и `codex`

## FAQ

**Бот не отвечает?**
Подожди — AI-провайдер может думать 10-60 секунд. Если ответа нет больше 2 минут, используй `/cancel` и попробуй снова.

**Как сбросить контекст?**
Команда `/new`.

**Как сменить модель?**
Команда `/model <имя>`. Перед этим можно посмотреть текущий провайдер и его модели через `/status` и `/model`.

**Голосовые ответы не приходят?**
Включи голосовой режим: `/voice`.

**Фото не обрабатываются?**
На сервере должен быть установлен tesseract-ocr. Обратись к администратору.

**Ответ слишком длинный?**
Бот автоматически отправит `.md` файл с полным текстом, а в чат — краткую версию.

**Что значит лимит сообщений?**
Администратор может ограничить количество сообщений в день. Проверь свой лимит: `/usage`.
