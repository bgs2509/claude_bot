# Исследование: Skills для claude_bot

**Дата:** 2026-03-10
**Тема:** Анализ библиотек скиллов и их применимость для обработки сообщений пользователей

---

## 1. Что такое Skills в Claude Code

**Skills** (навыки) — это расширяемые команды, которые добавляют Claude Code специализированные возможности. По сути это заранее написанные промпты/инструкции, которые активируются через `/имя`.

### Как работает

1. Пользователь пишет `/commit`, `/review-pr` и т.д.
2. Claude Code находит соответствующий skill
3. Skill разворачивается в полный промпт с детальными инструкциями
4. Claude выполняет задачу по этим инструкциям

### Типы

- **Встроенные** — идут с Claude Code из коробки (например `/commit`)
- **Пользовательские** — можно создавать свои в настройках проекта
- **От MCP-серверов** — могут добавляться внешними инструментами

### Отличие от CLI-команд

- `/help`, `/clear`, `/compact` — **встроенные CLI-команды** (обрабатываются приложением)
- `/commit`, `/review-pr` — **skills** (обрабатываются через инструмент `Skill`)

## 2. Доступные skills для разработки проекта

| Skill | Назначение | Релевантность |
|-------|-----------|---------------|
| `/commit` | Анализ изменений, формирование сообщения коммита | Высокая — используется постоянно |
| `/review-pr` | Code review pull request'а | Высокая — перед мержем важных изменений |

## 3. Идеи skills внутри проекта для обработки сообщений

### 3.1 `/review` — ревью кода из сообщения

**Где:** `handlers/text.py` или `handlers/document.py`

Пользователь кидает файл `.py` / `.js` → бот формирует промпт не просто "вот файл", а с инструкцией провести code review: найти баги, проблемы безопасности, предложить улучшения.

### 3.2 `/explain` — объяснение кода

**Где:** `handlers/document.py`

Пользователь загружает файл с подписью `/explain` → бот просит Claude объяснить код простым языком.

### 3.3 `/fix` — исправление ошибки

**Где:** `handlers/text.py`

Пользователь кидает traceback или лог ошибки → бот распознаёт это (по паттернам `Traceback`, `Error`, `Exception`) и автоматически формирует промпт "проанализируй ошибку и предложи исправление".

### 3.4 `/summarize` — сводка длинного текста

**Где:** `handlers/document.py`, `handlers/photo.py`

Пользователь кидает длинный документ или фото документа → бот делает сводку в 5-10 пунктов. Особенно полезно в связке с OCR.

### 3.5 `/translate` — перевод

**Где:** `handlers/text.py`, `handlers/photo.py`

Текст или фото с иностранным текстом → OCR + промпт на перевод.

### 3.6 `/commit` — генерация коммита в проекте пользователя

**Где:** `handlers/text.py`

Пользователь пишет `/commit` → бот вызывает Claude CLI с промптом, который анализирует `git diff` в `project_dir` пользователя и коммитит.

### Архитектура маршрутизации

```
[Текст пользователя]
    ↓
[Детектор intent'а]  — по префиксу (/review, /fix) или автоматически
    ↓
[Обогащение промпта] — добавить system-инструкцию под конкретный skill
    ↓
[run_claude()]
```

Не потребует изменения `services/claude.py` — только обёртка в хендлерах.

## 4. Исследование библиотек скиллов/промптов

### 4.1. Топ-3: максимально полезные

#### `anthropics/skills` (~87k stars)

- **GitHub:** https://github.com/anthropics/skills
- **Что предоставляет:** Официальный стандарт SKILL.md от Anthropic. Папки с инструкциями, скриптами и ресурсами, которые Claude загружает динамически. Скиллы для создания PDF, Excel, PowerPoint, Word, алгоритмического арта.
- **Python-совместимость:** Да, скиллы — это Markdown-файлы с YAML-метаданными.
- **Извлечение как standalone промптов:** Отлично. Каждый скилл — это SKILL.md файл. Формат: директория с SKILL.md + опциональные scripts/, references/, assets/. Можно напрямую читать и передавать содержимое в Claude CLI.
- **Применимость:** Идеальное совпадение с архитектурой бота — нативный формат для Claude CLI.

#### `f/awesome-chatgpt-prompts` (~144k stars)

- **GitHub:** https://github.com/f/awesome-chatgpt-prompts
- **Что предоставляет:** Крупнейшая коллекция ролевых промптов. Роли: переводчик, рецензент кода, Linux-терминал, писатель, консультант, аналитик. Файл `prompts.csv` для программного доступа.
- **Python-совместимость:** Да, чистый текст в CSV/Markdown.
- **Извлечение:** Отлично. Промпты — текстовые строки "Act as..." для prepend к сообщению.
- **Применимость:** Пользователь пишет `/role переводчик` → бот достаёт промпт из CSV → prepend → Claude CLI.

#### `langgptai/awesome-claude-prompts` (~4.2k stars)

- **GitHub:** https://github.com/langgptai/awesome-claude-prompts
- **Что предоставляет:** Промпты, оптимизированные для Claude: Code Review & Analysis, Architecture Planning, Technical Documentation, Debugging & Error Analysis, Performance Optimization.
- **Извлечение:** Отлично. Уже оптимизированы под Claude.
- **Применимость:** Готовые промпты, работающие лучше именно с Claude.

### 4.2. Средний уровень: полезны для конкретных задач

#### `abilzerian/LLM-Prompt-Library` (~772 stars)

- **GitHub:** https://github.com/abilzerian/LLM-Prompt-Library
- **Что предоставляет:** 164+ Jinja2-шаблонов по доменам: Finance, Legal, Medical, Programming, AI Research, Governance, Writing/Editing.
- **Извлечение:** Хорошо. Jinja2-шаблоны рендерятся Python'ом, результат → Claude CLI. Требует `jinja2`.
- **Применимость:** Для специализированных областей пользователей.

#### `masci/banks` (~122 stars)

- **GitHub:** https://github.com/masci/banks
- **Что предоставляет:** Лёгкий менеджер промптов на Jinja2. Хранение на диске с метаданными, управление версиями, модульная композиция.
- **Извлечение:** Хорошо. Рендерит шаблоны в текст → Claude CLI. Минимум зависимостей.
- **Применимость:** Движок для хранения и подстановки скиллов в боте.

#### `ai-prompter` (PyPI)

- **PyPI:** https://pypi.org/project/ai-prompter/
- **Что предоставляет:** Рендеринг Jinja2-шаблонов из файлов. Модульная композиция через `include`. Авто-переменная `current_time`.
- **Извлечение:** Хорошо. `pip install ai-prompter`, минимальные зависимости.
- **Применимость:** Простой менеджер скиллов.

#### `promptslab/Promptify` (~4.6k stars)

- **GitHub:** https://github.com/promptslab/Promptify
- **Что предоставляет:** Шаблоны для NLP: NER, Text Classification, QA, Relation Extraction, Tabular data. Структурированный вывод (JSON, списки).
- **Извлечение:** Средне. Ориентирована на свой API, но Jinja2-шаблоны можно извлечь из `promptify/prompts/`.
- **Применимость:** Для NLP-задач — извлечение сущностей, классификация.

### 4.3. Образовательные: для улучшения качества промптов

#### `0xeb/TheBigPromptLibrary` (~4.2k stars)

- **GitHub:** https://github.com/0xeb/TheBigPromptLibrary
- **Что предоставляет:** Реальные системные промпты от ChatGPT, Claude, Gemini, Copilot. Образовательный ресурс.
- **Применимость:** Для улучшения `--append-system-prompt` в `run_claude()`.

#### `NirDiamant/Prompt_Engineering` (~6.9k stars)

- **GitHub:** https://github.com/NirDiamant/Prompt_Engineering
- **Что предоставляет:** 22 туториала: Chain-of-Thought, Few-Shot, Self-Consistency, Role Prompting.
- **Применимость:** Справочник техник для построения качественных промптов.

#### `microsoft/semantic-kernel` (~27k stars)

- **GitHub:** https://github.com/microsoft/semantic-kernel
- **Что предоставляет:** SDK с Prompt Plugins — файлы `skprompt.txt` для суммаризации, перевода, чатов.
- **Извлечение:** Средне. Можно вытащить тексты из `samples/plugins/`. Сам SDK тяжёлый.
- **Применимость:** Источник промптов, не как зависимость.

## 6. Рекомендация по приоритету внедрения

| Приоритет | Библиотека | Зачем |
|-----------|-----------|-------|
| 1 | `anthropics/skills` | Формат хранения скиллов (SKILL.md) — нативный для Claude |
| 2 | `awesome-claude-prompts` | Готовые промпты, оптимизированные под Claude |
| 3 | `banks` или `ai-prompter` | Движок рендеринга Jinja2-шаблонов с переменными |
| 4 | `awesome-chatgpt-prompts` | Ролевые промпты (CSV) — большая коллекция |

### Пример интеграции

```
Пользователь: /skill review
Бот: загружает skills/review/SKILL.md
     → prepend к сообщению пользователя
     → run_claude(skill_prompt + user_text)
     → ответ Claude в роли code reviewer
```

### Ключевой вывод

Для проекта, работающего через Claude CLI subprocess, наибольшую ценность представляют библиотеки, где скиллы — это **структурированные текстовые промпты/шаблоны**, которые можно извлечь и передать как текст в CLI. Формат `anthropics/skills` (SKILL.md) — идеальный выбор, так как он нативен для Claude и поддерживается стандартом agentskills.io.
