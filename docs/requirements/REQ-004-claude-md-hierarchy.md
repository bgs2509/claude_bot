## Функциональные требования

### FR-1: Четырёхуровневая иерархия
- Уровень 0: `~/.claude/CLAUDE.md` — глобальные правила (язык, безопасность, инструментарий)
- Уровень 1: `ai-steward/CLAUDE.md` — правила бота (planner, шаблоны, onboarding)
- Уровень 2: `{User}/CLAUDE.md` — профиль юзера (имя, возраст, стиль общения, проекты)
- Уровень 3: `{User}/{Project}/CLAUDE.md` — правила проекта (зависят от типа)

### FR-2: Dev-правила в отдельном rules-файле
- `~/.claude/rules/python-dev.md` — стиль кода, git, окружение, тестирование, зависимости

### FR-3: Профили пользователей
- Каждый юзер имеет CLAUDE.md с профилем, стилем общения, списком проектов
- Стиль общения включает персонажа (выбор юзера или авто-адаптация)

### FR-4: Шаблоны проектов
- 10 типов: Health, Health-Lite, Investment, Budget, Family, Study, Career, Home, Hobby, Dev
- Шаблоны хранятся в templates/ в репо бота

### FR-5: Автообновление
- Каждый уровень CLAUDE.md содержит секцию автообновления
- При изменениях — автоматическое обновление соответствующего файла

### FR-6: Мульти-машинность
- Глобальный CLAUDE.md идентичен на всех машинах
- Машино-специфичное — в settings.local.json

### FR-7: Адаптация к пользователю
- Каждый уровень содержит правило адаптации, детализация нарастает к нижним уровням

## Нефункциональные требования

### NFR-1: Совместимость
- Claude Code загружает все CLAUDE.md от CWD вверх автоматически (без git)

### NFR-2: Не трогать существующие файлы
- Health/CLAUDE.md и investment/CLAUDE.md на VPS — не изменять
