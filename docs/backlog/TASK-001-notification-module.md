# TASK-001: Модуль уведомлений (Этап 1 из ROADMAP)

## Статус

New

## Описание

Реализовать систему персональных напоминаний для пользователей Telegram-бота. В каждом проекте (включая `__global__`) хранится файл `notify.json` с уведомлениями. Фоновая asyncio-задача каждую минуту сканирует все файлы уведомлений и рассылает напоминания в Telegram каждому получателю из списка `recipients`. Пользователь может создавать, редактировать и удалять уведомления через диалог с Claude в чате, а также просматривать активные через команду `/notify`.

## Требования

### Модель данных (notify.json)

Каждое уведомление содержит поля:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | uuid4 | Уникальный идентификатор |
| `title` | str | Заголовок уведомления |
| `description` | str | Подробное описание |
| `category` | enum | `health` / `event` / `todo` / `reminder` |
| `datetime` | ISO 8601 | Дата и время срабатывания |
| `remind_before` | list[int] | Список минут до события (напр. `[60, 10]`) |
| `repeat` | str | Правило повторения: `none` / `daily` / `weekly` / `monthly` |
| `recipients` | list[int] | Список Telegram user ID получателей |
| `status` | enum | `pending` / `sent` / `done` |
| `created_at` | ISO 8601 | Дата создания |
| `created_by` | int | Telegram user ID создателя |
| `sent_reminders` | list[int] | Отправленные remind_before (во избежание дублей) |

### Хранение

- Файл `notify.json` в директории каждого проекта (рядом с `session.json`)
- Включая проект `__global__`

### Фоновая задача

- asyncio-задача, запускается при старте бота
- Интервал проверки: `NOTIFY_SCAN_INTERVAL` (секунды, из `.env`, default 60)
- Сканирует все `notify.json` всех проектов
- Отправляет напоминание когда: `now >= datetime - remind_before_minutes`
- Фиксирует отправленные remind_before в `sent_reminders` (дедупликация)
- Обновляет `status` → `sent` после первого напоминания, `done` после события

### Повторяющиеся уведомления

- `daily` — сдвигает `datetime` на +1 день после `done`, сбрасывает `status` → `pending`
- `weekly` — сдвигает на +7 дней
- `monthly` — сдвигает на +1 месяц (calendar)

### Команда /notify

- Показывает список активных уведомлений текущего пользователя (`status: pending | sent`)
- Пагинация если уведомлений много
- Показывает: title, category, datetime, remind_before

### Создание / редактирование / удаление

- Через диалог с Claude в чате (инструкции на естественном языке)
- Claude формирует JSON-запись и записывает в `notify.json` проекта
- Удаление — установка `status: done` или полное удаление записи

### Конфигурация

- `NOTIFY_SCAN_INTERVAL` — интервал сканирования в секундах (default: `60`)
