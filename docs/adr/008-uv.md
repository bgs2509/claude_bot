# ADR-008: uv как пакетный менеджер

**Статус:** Принято
**Дата:** 2026-03-06

## Контекст

Проект использует `pyproject.toml` для метаданных и зависимостей. Нужен пакетный менеджер для установки, lock-файлов и запуска. Ранее использовался `pip` + `requirements.txt`.

## Решение

uv — быстрый пакетный менеджер для Python (замена pip/pip-tools/virtualenv).

Ключевые аргументы:
- **Скорость** — установка зависимостей в 10-100 раз быстрее pip. Критично для деплоя и CI.
- **Lock-файл** — `uv.lock` фиксирует точные версии всех зависимостей.
- **Всё в одном** — создание venv, установка, запуск (`uv run claude-bot`) без отдельного `virtualenv`.
- **Совместимость** — читает стандартный `pyproject.toml`, не требует специального формата.
- **Makefile** — `make install` → `uv sync`, `make run` → `uv run claude-bot`.

## Альтернативы

- **pip + venv** — стандартный, но медленный, нет встроенного lock-файла.
- **Poetry** — lock-файл и venv, но медленнее uv, нестандартный формат `[tool.poetry]`.
- **pdm** — поддерживает PEP 621, но меньше сообщество и медленнее uv.
