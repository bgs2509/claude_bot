# ADR-002: Claude Code CLI вместо API

**Статус:** Принято
**Дата:** 2026-03-06

## Контекст

Бот должен давать пользователю доступ к Claude с возможностью работы с файловой системой, выполнения команд и использования MCP-серверов (GitHub, Playwright, Brave Search). Нужен выбор: вызывать Claude через API или через CLI.

## Решение

Claude Code CLI (`claude -p ...`) вызывается как subprocess с `--output-format json`.

Ключевые аргументы:
- **Доступ к файловой системе** — Claude читает/пишет файлы в директории проекта (`cwd`). API не имеет доступа к ФС.
- **MCP-серверы** — CLI автоматически подключает настроенные MCP (GitHub, Docker, Playwright). Через API пришлось бы реализовывать tool use вручную.
- **Сессии на диске** — `--resume session_id` продолжает диалог с полным контекстом. Не нужно хранить историю сообщений.
- **Управление правами** — `--dangerously-skip-permissions` для admin/user, без флага для readonly.

Недостатки:
- Нет поддержки изображений (обходим через OCR + текстовое описание).
- Медленнее API из-за инициализации CLI на каждый запрос.

## Альтернативы

- **Anthropic API** — быстрее, поддержка vision, но нет доступа к ФС и MCP. Потребовалось бы реализовывать tool use для файлов.
- **Claude Code SDK** — программный интерфейс к CLI, но на момент решения не было стабильного Python SDK.
