# Architecture Decision Records (ADR)

Архитектурные решения проекта claude_bot.

## Формат

Каждый ADR содержит три секции:

1. **Контекст** — проблема и условия
2. **Решение** — что выбрали и почему
3. **Альтернативы** — что рассматривали и почему отказались

Статусы: **Принято** | **Заменено**

## Шаблон нового ADR

```markdown
# ADR-NNN: Название решения

**Статус:** Принято
**Дата:** YYYY-MM-DD

## Контекст

Описание проблемы и условий.

## Решение

Что выбрали и ключевые аргументы.

## Альтернативы

- **Вариант A** — причина отказа.
- **Вариант B** — причина отказа.
```
