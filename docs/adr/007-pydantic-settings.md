# ADR-007: pydantic-settings для конфигурации

**Статус:** Принято
**Дата:** 2026-03-06

## Контекст

Бот имеет 10+ параметров конфигурации: токен, пути, модели Whisper, голоса TTS, лимиты пользователей. Нужна валидация при старте и типобезопасный доступ к настройкам.

## Решение

pydantic-settings 2.7 — `Settings(BaseSettings)` с загрузкой из `.env`.

Ключевые аргументы:
- **Валидация при старте** — ошибка в `.env` ловится сразу, а не при первом использовании.
- **Типизация** — `Path`, `int`, `str`, `dict` с автоматическим приведением.
- **JSON-поля** — `USERS` хранится как JSON-строка в `.env`, парсится через `@field_validator`.
- **Значения по умолчанию** — `whisper_model="base"`, `claude_timeout=600` — можно запустить с минимальным `.env`.
- **Интеграция** — объект `settings` пробрасывается через middleware в каждый хендлер.

## Альтернативы

- **python-dotenv + os.getenv()** — нет валидации, нет типов, ошибки только в runtime.
- **dynaconf** — мощнее (profiles, secrets), но избыточно. Больше зависимостей.
- **YAML/TOML конфиг** — не интегрируется с переменными окружения для деплоя (systemd, Docker).
