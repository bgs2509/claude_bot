# ADR-004: edge-tts для TTS

**Статус:** Принято
**Дата:** 2026-03-06

## Контекст

Бот должен озвучивать ответы Claude голосом (режим `/voice`). Нужен TTS с качественными русскими голосами и низкой задержкой.

## Решение

edge-tts 7.2.7 — обёртка над Microsoft Edge Read Aloud API.

Ключевые аргументы:
- **Бесплатный** — использует публичный API Microsoft Edge без ключей и регистрации.
- **Качественные нейросетевые голоса** — `ru-RU-DmitryNeural` (мужской), `ru-RU-SvetlanaNeural` (женский).
- **Асинхронный** — `await communicate.save(path)` интегрируется в asyncio-цикл бота.
- **Простой API** — 3 строки кода: создать Communicate, вызвать save, отправить файл.

Ограничение: лимит 3000 символов на запрос (обрезается в коде).

## Альтернативы

- **gTTS (Google TTS)** — бесплатный, но роботизированный голос, не нейросетевой.
- **Silero TTS** — локальный, но требует ~500 МБ модели, добавляет нагрузку на CPU.
- **Yandex SpeechKit** — качественный, но платный и требует API-ключ.
- **Amazon Polly** — качественный, но платный, требует AWS аккаунт.
