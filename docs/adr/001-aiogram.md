# ADR-001: aiogram 3.15 как Telegram-фреймворк

**Статус:** Принято
**Дата:** 2026-03-06

## Контекст

Нужен Python-фреймворк для Telegram Bot API. Бот обрабатывает текст, голос, фото и документы, требуется поддержка FSM, middleware и inline-клавиатур. Все I/O-операции (Claude CLI, STT, TTS) асинхронные.

## Решение

aiogram 3.15 — асинхронный фреймворк на asyncio.

Ключевые аргументы:
- **Нативный asyncio** — бот вызывает `asyncio.create_subprocess_exec` для Claude CLI, `edge-tts` и `ffmpeg`. Синхронный фреймворк потребовал бы отдельных потоков.
- **Встроенный FSM** — используется для создания проектов (`CreateProject.waiting_name`).
- **Middleware** — `AuthMiddleware` проверяет доступ и пробрасывает зависимости в каждый хендлер.
- **Роутеры** — каждый тип сообщений в отдельном роутере (`text`, `voice`, `photo`, `document`, `menu`).
- **Активное сообщество** — русскоязычная документация, частые обновления.

## Альтернативы

- **python-telegram-bot** — синхронный по умолчанию, async-режим экспериментальный. Менее удобная система middleware.
- **Telethon** — MTProto, избыточен для бота. Ориентирован на userbot, не на Bot API.
- **pyrogram** — хороший вариант, но меньше сообщество и нет встроенного FSM.
